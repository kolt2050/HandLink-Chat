# Group HandLink Chat
Прототип мультипользовательского браузерного расширения для зашифрованного peer-to-peer чата без собственного backend-сервера.
- Использует публичный STUN-сервер Google `stun:stun.l.google.com:19302` только для WebRTC NAT traversal.
- Позволяет создать именованную комнату чата.
- Генерирует invite+offer пакет для приглашения другого пользователя.
- Позволяет пригласить нового пользователя в текущую комнату.
- Принимает invite+offer и answer пакеты от других пользователей.
- Устанавливает прямые WebRTC data-channel соединения между браузерами.
- Шифрует сообщения общим секретом комнаты из invite.
- Сохраняет историю сообщений локально в браузере через IndexedDB, до 1000 последних сообщений на чат.
- Восстанавливает последнюю комнату, имя комнаты, nickname и сохранённые сообщения после обновления страницы.
- Может держать несколько ручных WebRTC peer-соединений одновременно.
- Пересылает зашифрованные сообщения между подключёнными участниками.
- Позволяет цепочке вроде пользователь 1 - пользователь 2 - пользователь 3 синхронизировать сообщения через relay.
- Защищается от циклического повторного пересылания одинаковых encrypted payload.
- Содержит сворачиваемый блок Live Logs для отладки соединений.
- Логи показывают создание offer/answer, открытие и закрытие WebRTC-каналов, relay, синхронизацию истории и ошибки.
- Не использует собственный backend-сервер.
- Не реализует автоматический public discovery, увы.
- В качестве signaling-пути использует ручной обмен invite+offer и answer пакетами.
- Это прототип.
